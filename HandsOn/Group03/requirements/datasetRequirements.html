Dataset requirements

Requirements satisfied

  • R1. Data are in one of the selected smart city domains.

    The dataset shows the quality of air in Madrid, which is one of the available domain to be covered

  • R2. Data are available as a CSV file.

    Every file in the dataset has CSV format.

  • R3. Data have an open license so they can be published.

    The dataset has an open license, has written below

  • R4. Data can be easily linked with generic real-world entities (e.g. locations).

    The dataset has many entities that can be linked, as written below.

Source and license

All the data present in this dataset comes from "Madrid's City Council Open Data" website (link).
The data therefore inherits the "Madrid Open Data Terms of Use" (link) to the spanish detailed version of the license, which allow for free commercial and non-commercial use, and provide no liability on the data.

Entities that can be linked with other datasets

  • stations

    Latitude and Longitude
    Position of stations that collect measurements of pollutants. They can be linked to a dataset of places in Madrid

  • csvs_per_year

    Date
    Timestamp of the measurements. Together with the station's position can be linked to other parameters (i.e. temperature, humidity, etc.)

Other dataset that can be linked to the entities

We can use information that can be correlated to the position of the measurements

  • Temperature
  • Pressure
  • Humidity
  • Wind
