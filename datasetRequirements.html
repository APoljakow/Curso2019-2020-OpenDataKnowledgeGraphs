Dataset requirements

Requirements satisfied

  • R1. Data is in the smart city domain.

    The dataset shows the wi-fi spot distribution in New York city, which is one of the available domains to be covered.

  • R2. Data is available as a CSV file.

    Every file in the dataset has a CSV format.

  • R3. Data has an open license so it can be published.

    The dataset has an open license, as it's stated below.

  • R4. Data can be easily linked with generic real-world entities (e.g., locations).

    The dataset has many entities that can be linked, as it's stated below.

License

All the data present in this dataset comes from "New York City Open Data" website (link).
The data is regulated according to the "New York City Open Data Terms of Use" (link) , which allow for free commercial and non-commercial use, and provide no liability on the data.
This dataset is also available in Kaggle (link) , where the type of license is stated (CC0: Public Domain).

Entities that can be linked with other datasets

  • ()

    Locations

    Latitude and Longitude
    Positions of the wi-fi spots. They can be linked to a dataset of places(boroughs and zipcodes) in New York City.

  • Provider

    Carrier ID
    Connection carriers of the wi-fi spots. Together with the spot's position can be linked to other parameters (i.e. intensity, ratings, cost)
